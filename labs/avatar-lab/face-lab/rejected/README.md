# rejected/ — rostos reprovados

Rostos que **falharam um portão** ou ficaram **abaixo do piso** (`Overall < 7.5` ou qualquer sub-nota `< 6`), segundo o `../FACE_SCORE_GUIDE.md`.

## Como entra aqui
- Status `REJECTED` gravado no `meta.json` **e** no `../FACE_RANKING.md`.
- **Sempre registrar o motivo** no `meta.json → notas` e no `review.md` (qual portão/critério falhou e **por quê**, com evidência).
- **Mover** a pasta `face_NNN/` inteira para cá.

## Por que guardar reprovados
- Reprovação é **informação útil**: mostra quais direções de prompt não funcionam, evitando repetir o erro na próxima leva.
- Faixa **7.5–8.4** (portões ok, mas abaixo de aprovado) = **base viável**, não vencedor: registrar como "base B" em `notas` antes de mover, conforme `../FACE_SCORE_GUIDE.md`.

> Inicia vazia.
